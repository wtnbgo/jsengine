# Demo 4 (Canvas2D アニメーション) 確認手順

`-demo 4` で起動、または起動後 `4` キーで Demo 4 に切替。
画面は **512 × 512**。毎フレーム時刻 `time` ms に依存して動く。

## ブラウザ側の「正解」リファレンス

`docs/demo4_reference.html` をブラウザで開くと、同じ描画コードをブラウザの Canvas 2D API で実行した結果が見られます。一時停止 / 1 フレーム進めるボタンを用意。

```
cd D:/test/jsengine
python -m http.server 8000
```
で `http://localhost:8000/docs/demo4_reference.html` を開いてください。
**`t` 値（左上に表示）を揃えてから両方を一時停止すると比較しやすい。**

## 描画内容（時刻 t に依存）

### 中央 — 6 つの回転する四角形（同心）

- 中心 (256,256) に配置、`translate→rotate→fillRect(-s/2,-s/2,s,s)`
- サイズ: `40 + i*15` (i=0..5) → 40, 55, 70, 85, 100, 115
- 回転速度: `time * 0.001 * (i+1) * 0.5 + i*π/3` → i が大きいほど速く回る
- 色相: `(i*60 + time*0.05) % 360` で時間と共に色相シフト
- α=0.7 で塗り、白線 lw=1 で外枠

### 中央 — 白い円（脈動）

- 半径 `30 + sin(time*0.002)*10` で 20px〜40px の間を脈動
- 中心 (256,256) に塗りつぶし白

### 軌道 — 8 つの小さな円

- 中心 (256,256) から半径 150 の円周上に等間隔 8 個 (π/4 ごと)
- 各円の角度: `time*0.001 + j*π/4`
- 半径 10、`rgba(255,255,255,0.5)` 半透明白

### テキスト（RobotoMono 18px）

- 左上 (10,30): `t=X.Xs` (経過秒)
- 左下 (10,500): `Demo 4: Animation`

## チェックポイント

- [ ] **回転方向**: すべての四角形が時計回り（外側ほど速い）
- [ ] **色相循環**: 数秒で全四角形の色が一巡（外側矩形は約 6.7 秒で 360°）
- [ ] **円脈動**: 中央の白円が約 π 秒（≈3.14 s）周期で 20→40→20 と脈打つ
- [ ] **軌道速度**: 8 つの小円が 2π 秒（≈6.28 s）で 1 周
- [ ] **α合成**: 軌道円が四角形に重なる部分で背景透過
- [ ] **フォント**: `t=X.Xs` が等幅で表示される（RobotoMono）

## よくある不具合パターン

- 回転中心がズレる → `translate` 後の `rotate` 順序ミス
- 色がフラッシュする → `floor(time*0.05)` 切り捨てが効きすぎている（jsengine/ブラウザ両方同じはず）
- 軌道円が消える → α=0.5 の合成が機能していない
- 等幅にならない → RobotoMono の解決失敗 → 別フォントに fallback

## jsengine 固有の注意

- `time` は jsengine 側では `SDL_GetTicks` ベースのミリ秒、ブラウザは `performance.now()` ベース。両者を同じ時刻に揃えるには一時停止ボタンで `t` 値（左上）を見比べる必要がある。
- Canvas2D のテキストは ThorVG FT loader 経由。等幅性はフォント側の `advance` に依存する。
