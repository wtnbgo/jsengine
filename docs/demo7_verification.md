画面は **640 × 480**。リサジュー軌道上を動く黄色矩形 + 初回のみ描画される背景。

## 描画内容

### 背景（初回フレームのみ）

- **縦縞グラデーション**: Y=0..480 を 4px ごとに `rgba(r, 50, 255-r, 1)` (r は y に比例) で塗る
- **グリッド線**: x=0,40,80,... と y=0,40,80,... に `rgba(255,255,255,0.2)` 線幅 1
- **タイトル**: (10,30) `Demo 7: Dirty Rect Update` 20px OpenSans-Bold 白
- **説明**: (10,55) `Background drawn once, only moving rect updates each frame` 14px OpenSans-Regular 白

### 毎フレーム — 黄色矩形がリサジュー軌道で動く

- 位置: `(300 + cos(t)*200, 240 + sin(t*1.3)*150)`
- サイズ: 60×60
- 塗り: `rgba(255,255,0,0.9)` ほぼ不透明黄
- 外枠: 白 lw=2

### 前フレーム位置の処理

- `clearRect(prevX-2, prevY-2, 64, 64)` で前回位置を消す
- 同じ領域に背景縞を 4px 単位で再描画
- → ブラウザ側でも jsengine 側でも同じ「消して塗り直す」流れ

## チェックポイント

- [ ] **背景の縞**: 上端は青寄り (`rgba(0,50,255)`)、下端は赤寄り (`rgba(255,50,0)`)、中央 (y=240) は紫
- [ ] **グリッド**: 40px 間隔で薄い白線が全画面に格子状
- [ ] **タイトル文字**: ボールド、`Dirty Rect Update` がはっきり見える
- [ ] **リサジュー軌道**: 矩形が 2π 秒 (≈6.28 s) で x 方向 1 周、`2π/1.3` 秒 (≈4.83 s) で y 方向 1 周
- [ ] **継ぎ目なし**: 矩形が通過した跡に背景がそのまま戻る（縞の位置・色がズレない）
- [ ] **黄色の透明感**: α=0.9 なので背景がうっすら透けて見える
- [ ] **白い外枠**: 線幅 2 で内側にジャギーが無い

## よくある不具合パターン

- **矩形の軌跡が残る**: `clearRect` 後の背景再描画範囲がズレている
- **背景の縞が二重になる**: 再描画範囲が広すぎて隣接の縞も塗り直されている
- **黄色矩形が真っ黄色**: α=0.9 の合成がされず不透明として描かれている
- **タイトルが消える**: タイトル描画位置 (Y=30) と矩形軌道 (Y=90〜390) の重なりでクリアされている
- **dirty rect 最適化のバグ** (jsengine 固有): 部分テクスチャアップロードの矩形がズレて、画面の一部に古い内容が残る

## jsengine 固有の注意

- jsengine は dirty rect 領域だけを GL テクスチャに部分アップロードする最適化を持つ。`flush()` は `drawCanvas2DAt` 内の texture getter で自動実行される。
- 内部最適化のバグはブラウザでは出ない — ブラウザは毎フレーム全面 redraw なので「見た目だけ」を比較する。
- 黄色矩形が「軌跡を引いて流れる」ように見えたら dirty rect バグの可能性が高い（clearRect の範囲が狭すぎ）。
- 逆に「画面の一部に白いゴースト」が残るなら、テクスチャアップロード範囲のオフセット計算ミス。
